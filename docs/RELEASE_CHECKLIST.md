# Release Checklist

Use this checklist before creating a public release of a game or this template.

## Repository

- [ ] README is up to date.
- [ ] Project name is correct.
- [ ] License file exists.
- [ ] Credits are complete.
- [ ] Changelog is updated.
- [ ] No temporary files are committed.
- [ ] No export credentials are committed.

## Godot project

- [ ] Project opens in the target Godot version.
- [ ] Main scene runs.
- [ ] Autoload paths are valid.
- [ ] No missing scripts.
- [ ] No missing scene references.
- [ ] No missing resources in the demo scene.

## Gameplay

- [ ] Player can move.
- [ ] Player can interact with objects.
- [ ] Player can complete the demo objective.
- [ ] Flashlight works.
- [ ] UI prompts are readable.
- [ ] The game can be exited safely.

## Audio and visual

- [ ] No copyrighted audio is used without permission.
- [ ] Placeholder assets are marked as placeholders.
- [ ] Volume levels are acceptable.
- [ ] Visual effects do not make the game unreadable.

## Export

- [ ] Export templates are installed locally.
- [ ] Export preset exists for the target platform.
- [ ] Build folder is ignored by git.
- [ ] Windows build was tested if released.
- [ ] Linux build was tested if released.
- [ ] Web build was tested if released.

## Public page

- [ ] Screenshots are current.
- [ ] Description matches the actual game.
- [ ] Controls are listed.
- [ ] Known issues are listed.
- [ ] Credits and license notes are included.
