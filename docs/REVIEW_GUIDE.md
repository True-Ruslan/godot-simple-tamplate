# Review Guide

Use this guide when reviewing pull requests.

## Documentation PR

Check:

- Is the document useful?
- Is it easy to follow?
- Does it duplicate another file?
- Does it link to the right next step?

## Code PR

Check:

- Does the project still open?
- Does the main scene still run?
- Are scene paths valid?
- Are autoloads valid?
- Is the change small enough?

## Asset PR

Check:

- Is the license clear?
- Is the source URL recorded?
- Is attribution required?
- Is the file size reasonable?
- Is `docs/CREDITS.md` updated?

## Merge rule

If a PR makes the demo not runnable, do not merge it until the issue is fixed or clearly documented.
