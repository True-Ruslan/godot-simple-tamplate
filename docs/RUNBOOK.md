# Runbook

## When starting work

1. Read `docs/START_HERE.md`.
2. Check `docs/PROJECT_STATUS.md`.
3. Check open PRs.
4. Choose one small task.

## When something breaks

1. Stop adding features.
2. Read `docs/GODOT_PROJECT_AUDIT.md`.
3. Fix the smallest broken reference.
4. Run the smoke test.
5. Update changelog.

## Before release

1. Complete local validation.
2. Complete release checklist.
3. Review known limitations.
4. Update release notes.
5. Create tag only after validation.
