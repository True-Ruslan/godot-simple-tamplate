# Final Docs Pass

Before the first release, do a final documentation pass.

## Check

- README points to `docs/START_HERE.md`.
- README points to `docs/DOCUMENTATION_INDEX.md`.
- Repository clone URL is correct.
- Godot version is consistent across docs.
- Release notes match actual project state.
- Known limitations are honest.
- No docs promise features that are not implemented.

## Important

Do not market the template as production-ready until the project has been locally validated in Godot.
