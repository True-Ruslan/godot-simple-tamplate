# Workflow Overview

## Human workflow

1. Read `docs/START_HERE.md`.
2. Run the demo.
3. Pick a blueprint.
4. Define a playable slice.
5. Make one small change.
6. Playtest.
7. Repeat.

## AI-agent workflow

1. Read `AGENTS.md`.
2. Read `docs/AGENT_WORKFLOW.md`.
3. Run one prompt from `docs/PROMPT_ORDER.md`.
4. Make a small change.
5. Update docs.
6. Report manual test steps.

## Maintainer workflow

1. Review PR scope.
2. Check quality checklist.
3. Merge small PRs.
4. Validate locally before release.
5. Tag release only after smoke test.
