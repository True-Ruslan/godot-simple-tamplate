# User Journey

This document describes the intended user journey.

## New user

1. Finds the repository.
2. Reads README.
3. Opens `docs/START_HERE.md`.
4. Runs the demo scene.
5. Chooses a blueprint.
6. Creates a scenario.
7. Creates a playable slice.
8. Uses prompts with an AI agent.
9. Playtests.
10. Exports later.

## AI-assisted user

1. Gives the agent `AGENTS.md`.
2. Runs repository review prompt.
3. Runs template conversion prompt.
4. Runs playable slice prompt.
5. Makes one small change per iteration.
6. Reviews and tests after each change.

## Maintainer

1. Keeps docs consistent.
2. Reviews small PRs.
3. Validates Godot project locally.
4. Tags releases after validation.
