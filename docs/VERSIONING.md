# Versioning

This template uses simple semantic versioning.

## Format

```text
MAJOR.MINOR.PATCH
```

Example:

```text
v0.1.0
```

## Patch version

Use patch releases for:

- typo fixes;
- small documentation fixes;
- tiny script fixes;
- broken link fixes.

## Minor version

Use minor releases for:

- new example scenes;
- new template systems;
- new prompt packs;
- new blueprint groups;
- new export presets.

## Major version

Use major releases for breaking changes:

- folder structure redesign;
- changing core scene paths;
- changing required Godot major version;
- replacing the base architecture.

## Current target

The first stable target is:

```text
v0.1.0
```

It should mean: the repository is cloneable, documented, and the demo scene can be opened and tested in Godot.
