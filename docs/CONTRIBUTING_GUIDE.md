# Contributing Guide

## Principles

- Keep the template simple.
- Keep the demo scene runnable.
- Prefer small changes.
- Update documentation with code changes.
- Record external resources in credits.

## Before a pull request

1. Open the project in Godot.
2. Run the main scene.
3. Check changed scene paths.
4. Update docs when needed.

## Good first tasks

- Improve documentation.
- Add blueprint examples.
- Add small interaction examples.
- Improve manual tests.
